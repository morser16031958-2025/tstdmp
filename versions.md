# История версий проекта ssdmp

### v0.52 (Текущая)
- **USB/UAS**: Улучшена совместимость с ASMedia (ASM1153E). Метод **0xEE** теперь полностью соответствует CDI (убраны лишние байты в буфере). Добавлена попытка альтернативного подкода для SMART.
- **Протоколы**: В Miniport IOCTL добавлена поддержка альтернативной сигнатуры `ATAPI   ` (8 пробелов), которую требуют некоторые драйверы UAS для выполнения ATA-команд.

### v0.51
- **USB/UAS**: Добавлен метод **ASMedia Vendor Specific (0xEE)** для обхода блокировок `UASPStor`. Это основной способ, который использует CDI для мостов ASMedia (ASM1153E и др.).
- **Протоколы**: Пересмотрен "водопад" для ASMedia: теперь сначала пробуется команда 0xEE, затем Miniport, и только потом стандартный SAT.
- **Интерфейс**: В `show_raw_identify` добавлен тест метода `_asmedia_ata_identify_device`.

### v0.50
- **Детекция**: Добавлена эвристика определения моста по строке модели (ASMedia, JMicron, Realtek), если VID/PID не найден в базе. Это исправляет проблему с ASM1153E.
- **USB/UAS**: Добавлены новые PID для мостов ASMedia (`0x1153`, `0x5106`).
- **Отладка**: Добавлен подробный вывод ошибок (WinError) для Miniport-пути при использовании флага `--debug`.

### v0.49
- **USB/UAS**: Исправлен расчет смещения данных (`data_offset`) для Miniport IOCTL. Теперь данные IDENTIFY/SMART извлекаются корректно из буфера `_SENDCMDOUT512`.
- **Протоколы**: Приоритет Miniport-пути для неопознанных USB-устройств (`unknown_usb`) теперь явно выше, чем у SAT16/SAT12, для обхода блокировок UASPStor.
- **Интерфейс**: В `show_raw_identify` результат теста Miniport теперь выводится всегда (даже если он не удался), для облегчения отладки.

### v0.48
- **USB/UAS**: Добавлена поддержка чтения SMART/IDENTIFY через **SCSI Miniport IOCTL** (`IOCTL_SCSI_MINIPORT`). Это позволяет обходить блокировки команд ATA Pass-Through (SAT) драйверами UAS (например, UASPStor).
- **WMI**: Реализовано получение `SCSIPort` и `SCSITargetId` через WMI для автоматического поиска Miniport-пути.
- **Протоколы**: В функции `_smart_identify` и `_smart_read_data` добавлен этап опроса через Miniport для неизвестных USB-устройств.
- **Интерфейс**: В `show_raw_identify` добавлен вывод результата теста `_miniport_smart_identify`.

### v0.47
- **USB/UAS**: Исправлены точки вызова `_detect_usb_bridge`, теперь корректно передается `disk_number` для VID/PID детекции.
- **Детекция**: Добавлена поддержка `bus_type=0` (UAS/SCSI) для автоматического определения USB-мостов.
- **Отчеты**: В `show_raw_identify` добавлены тесты SAT12 и JMicron (0xDF) с выводом в общий список методов.
- **Управление**: Добавлен глобальный флаг `--debug` в CLI. В коде жестко заданный `debug=True` заменен на динамический параметр.
- **Логика**: Финальный выбор данных в `show_raw_identify` теперь использует общую функцию `_smart_identify`.

### v0.46
- **USB/UAS**: Исправлено чтение SMART/IDENTIFY для USB-дисков.
- **Протоколы**: Реализована поддержка SAT12 (CDB 0xA1) в дополнение к SAT16.
- **Вендоры**: Добавлена поддержка JMicron Vendor Command (0xDF) для обхода ограничений мостов.
- **Детекция**: Реализовано определение USB VID/PID через WMI для автоматического подбора команд моста.
- **Отладка**: Добавлено логирование SCSI Sense байтов при ошибках (в режиме `--full-diag`).
- **Исправления**: Починен парсинг `STORAGE_DEVICE_DESCRIPTOR` (смещение `BusType` и абсолютные адреса строк).

### v0.45
- **Протоколы**: Добавлен метод `_scsi_log_sense_smart` для чтения SMART через SCSI LOG SENSE (Page 0x2F). Это расширяет возможности диагностики для дисков, видимых как SCSI.
- **Детекция**: Добавлены специфические правила для Walram и бюджетных SSD в `controllers.py`.
- **Логика**: Улучшен порядок опроса SMART-данных (Legacy Win API -> ATA PT -> SAT -> SCSI LOG SENSE).

### v0.43
- **Исправление ошибок**: Устранен `NameError` в `reports.py` (восстановлены потерянные импорты `_scsi_get_info` и `_scsi_get_serial_vpd_80`).
- **Стабильность**: Проект полностью перекомпилирован для проверки целостности связей.

### v0.42
- **Диагностика**: Реализован многоуровневый fallback в `show_full_diag` (если ATA IDENTIFY заблокирован, данные берутся из WMI/SCSI).
- **SMART**: Добавлена поддержка Legacy SMART API (`IOCTL_SMART_RCV_DRIVE_DATA`) в `ata.py` как альтернативный канал чтения.
- **Интерфейс**: В детальный отчет добавлено поле `Interface` (тип шины).

### v0.41
- **Экстракция данных**: Улучшен разбор `PNPDeviceID` для извлечения уникального ID из сложных строк (поиск самого длинного сегмента между `&`).
- **Форматирование**: Исправлена склейка названия модели (например, `WALRAM 1 28GB` -> `WALRAM 128GB`).

### v0.40
- **Интеллект**: Реализована "умная" фильтрация полей. Данные об объеме (GB/TB) теперь автоматически переносятся из поля прошивки в модель.
- **Очистка**: Добавлена принудительная очистка всех строк от непечатаемых ASCII-символов.

### v0.39
- **Протоколы**: Исправлены смещения (offsets) в `SCSI Standard Inquiry` для точного соответствия спецификации (байты 8-15, 16-31, 32-35).
- **Приоритеты**: Серийный номер для SCSI-дисков теперь запрашивается через выделенную страницу `VPD 0x80`.

### v0.38
- **Рефакторинг (Этап 2)**: Модуль `ata.py` разделен. Созданы `winapi.py` (системные вызовы) и `wmi.py` (PowerShell запросы).
- **Архитектура**: Введена многослойная структура CLI -> Reports -> Protocols -> WinAPI.

### v0.37
- **Расширение**: Добавлены функции чтения `PNPDeviceID` и `SCSI VPD Page 0x80` для работы с бюджетными контроллерами.

### v0.36
- **WMI**: Улучшена детекция типа шины через прямое обращение к свойству `InterfaceType` в PowerShell.
- **SN**: Добавлена базовая очистка и замена пустых серийных номеров на `?`.

### v0.35
- **Рефакторинг (Этап 1)**: Файл `cli.py` разделен. Созданы `controllers.py` (логика детекции железа) и `reports.py` (формирование отчетов).

### v0.34
- **Диагностика**: Улучшена функция `show_full_diag`, добавлен вывод состояния здоровья (Health Status) с пояснением причин.
- **SMART**: Добавлен мониторинг температуры и ошибок интерфейса (UDMA CRC).

### v0.33
- **Базовая версия**: Начало анализа проекта. Поддержка ATA Identify и стандартного SMART.
